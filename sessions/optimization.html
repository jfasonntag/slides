Transportation Problem

A chemical company has two factories F1,F2F_1,F_2 and a dozen retail outlets R1,‚Ä¶,R12R_1,\dots,R_{12}. Each factory ii can produce at most aia_i tons of output each week. Each retail outlet jj has a weekly demand of bjb_j tons per week. The cost of shipping from FiF_i to RjR_j is given by cijc_{ij}. How much of the product to ship from each factory to each outlet, minimize cost, and satisfy all constraints? let's call xijx_{ij} the number of tons shipped from ii to jj.

Figure 1.2 in (Nocedal and Wright 2006)

  • A mathematical formulation of this problem is min‚àëijcijxijsubject to‚àëj=112xij‚â§ai,i=1,2‚àëi=12xij‚â•bj,j=1,‚Ä¶,12xij‚â•0,i=1,2,j=1,‚Ä¶,12 \begin{align} \min \sum_{ij} c_{ij} x_{ij} \\ \text{subject to} \sum_{j=1}^{12} x_{ij} \leq a_i,\quad i=1,2 \\ \sum_{i=1}^2 x_{ij} \geq b_j,\quad j=1,\dots,12 \\ x_{ij} \geq 0, \quad i=1,2,j=1,\dots,12 \end{align}

  • This is called a linear programming problem, because both objective function and all constrains are linear.

  • With any of those being nonlinear, we would call this a non-linear problem.

Convexity

  • Convex problems are easier to solve.
  • What is convex?

A set S‚àà‚ÑùnS\in\mathbb{R}^n is convex if the straight line segment connecting any two points in SS lies entirely inside SS. A function ff is a convex function, if its domain SS is a convex set, and for any two points x,y‚ààSx,y \in S, we have that f(Œ±x+(1‚àíŒ±)y)‚â§Œ±f(x)+(1‚àíŒ±)f(y) f(\alpha x + (1-\alpha)y) \leq \alpha f(x) + (1-\alpha) f(y) for all Œ±‚àà[0,1]\alpha \in [0,1]

  • Simple instances of convex sets are the unit ball {y‚àà‚Ñùn,‚Äñy‚Äñ2‚â§1}\{y \in \mathbb{R}^n, \Vert y\Vert_2 \leq 1\}, and any set defined by linear equalities and inequalities.
  • convex Programming describes a special case of the introductory minimizatin problem where
    • the objective function is convex,
    • the equality constrains are linear, and
    • the inequality constraints are concave.

Optimization Algorithms

  • All of the algorithms we are going to see employ some kind of iterative proceedure.
  • They try to improve the value of the objective function over successive steps.
  • The way the algorithm goes about generating the next step is what distinguishes algorithms from one another.
    • Some algos only use the objective function
    • Some use both objective and gradients
    • Some add the Hessian
    • and many variants more

Desirable Features of any Algorithm

  • Robustness: We want good performance on a wide variety of problems in their class, and starting from all reasonable starting points.
  • Efficiency: They should be fast and not use an excessive amount of memory.
  • Accuracy: They should identify the solution with high precision.

A Word of Caution

  • You should not normally attempt to write a numerical optimizer for yourself.
  • Entire generations of Applied Mathematicians and other numerical pro's have worked on those topics before you, so you should use their work.
    • Any optimizer you could come up with is probably going to perform below par, and be highly likely to contain mistakes.
    • Don't reinvent the wheel.
  • That said, it's very important that we understand some basics about the main algorithms, because your task is to choose from the wide array of available ones.

Unconstrained Optimization: What is a solution?

  • A typical unconstrained optimization problem will look something like this: minxf(x),x‚àà‚Ñùn \min_x f(x),\quad x\in \mathbb{R}^n and where f:‚Ñùn‚Ü¶‚Ñùf : \mathbb{R}^n \mapsto \mathbb{R} is a smooth function.
  • In general, we would always like to find a global minimizer, i.e.
    a point x*x^* where f(x*)‚â§f(x)‚àÄxf(x^*) \leq f(x)\quad \forall x
  • Since our algorithm is not going to visit many points in ‚Ñùn\mathbb{R}^n (or so we hope), we can never be totally sure that we find a global optimizer.
  • Most optimizers can only find a local minimizer. That is a point
    x*x^* where f(x*)‚â§f(x)‚àÄx‚ààùí©f(x^*) \leq f(x)\quad \forall x \in \mathcal{N} where ùí©\mathcal{N} is a neighborhood around x*x^*.
  • Global minization can be very hard sometimes.

Rosenbrock Banana and Optim.jl

  • We will use Optim for the rest of this lecture.
  • We need to supply the objective function and - depending on the solution algorithm - the gradient and hessian as well.
function rosenbrock(x::Vector)
    return (1.0 - x[1])^2 + 100.0 * (x[2] - x[1]^2)^2
end

function rosenbrock_gradient!(x::Vector, storage::Vector)
    storage[1] = -2.0 * (1.0 - x[1]) - 400.0 * (x[2] - x[1]^2) * x[1]
    storage[2] = 200.0 * (x[2] - x[1]^2)
end

function rosenbrock_hessian!(x::Vector, storage::Matrix)
    storage[1, 1] = 2.0 - 400.0 * x[2] + 1200.0 * x[1]^2
    storage[1, 2] = -400.0 * x[1]
    storage[2, 1] = -400.0 * x[1]
    storage[2, 2] = 200.0
end

The Golden Ratio or Bracketing Search for 1D problems

  • A derivative-free method
  • a Bracketing method
    • find the local minimum of ff on [a,b][a,b]
    • select 2 interior points c,dc,d such that a<c<d<ba<c<d<b
      • f(c)‚â§f(d)‚üπf(c) \leq f(d) \implies min must lie in [a,d][a,d]. replace bb with dd, start again with [a,d][a,d]
      • f(c)>f(d)‚üπf(c) > f(d) \implies min must lie in [c,b][c,b]. replace aa with cc, start again with [c,b][c,b]
    • how to choose b,db,d though?
    • we want the length of the interval to be independent of whether we replace upper or lower bound
    • we want to reuse the non-replaced point from the previous iteration.
    • these imply the golden rule:
    • new point xi=a+Œ±i(b‚àía)x_i = a + \alpha_i (b-a), where Œ±1=3‚àí52,Œ±2=5‚àí12\alpha_1 = \frac{3-\sqrt{5}}{2},\alpha_2=\frac{\sqrt{5}-1}{2}
    • Œ±2\alpha_2 is known as the golden ratio, well known for it's role in renaissance art.

Bracketing for Multidimensional Problems: Nelder-Mead

  • The Goal here is to find the simplex containing the local minimizer x*x^*
  • In the case where ff is n-D, this simplex has n+1n+1 vertices
  • In the case where ff is 2-D, this simplex has 2+12+1 vertices, i.e. it's a triangle.
  • The method proceeds by evaluating the function at all n+1n+1 vertices, and by replacing the worst function value with a new guess.
  • this can be achieved by a sequence of moves:
    • reflect
    • expand
    • contract
    • shrink movements.
  • this is a very popular method. The matlab functions fmincon and fminsearch implements it.
  • When it works, it works quite fast.
  • No derivatives required.
using Optim
optimize(rosenbrock, [0.0, 0.0], method = :nelder_mead)
  • But.

Optimality Conditions

  • First Order Necessary Conditions: If ff is continously differentiable and x*x^* is a local minimizer of ff, then ‚àáf(x*)=0\nabla f(x^*) = 0.
  • Second Order Necessary Conditions: If ff is twice continuously differentiable and x*x^* is a local minimizer of ff, then ‚àáf(x*)=0\nabla f(x^*) = 0 and Hf(x*)H_f (x^*) is positive semi-definite, i.e. we have sTHf(x*)s‚â•0s^T H_f (x^*) s \geq 0 for all s‚àà‚Ñùns \in \mathbb{R}^n.
  • Second Order Sufficient Conditions: If ff is twice continuously differentiable and at x*x^* we have that ‚àáf(x*)=0\nabla f(x^*) = 0 and Hf(x*)H_f (x^*) is positive definite, i.e. sTHf(x*)s>0,s‚â†0s^T H_f (x^*) s > 0,s\neq 0 then x*x^* is a local minimizer of ff.
  • More sophisticated solvers make ample use of those.

Reminder: Sufficient Conditions for Global Optimality

  • If ff is convex, then any local minimizer x*x^* is a global minimizer.
  • If ff is convex and differentiable on ‚Ñùn\mathbb{R}^n , then any point x*x^* is a global minimizer if and only if it is a stationary point, i.e. if f‚àáf(x*)=0\nabla f(x^*) = 0.

Two Strategies: Line Search and Trust Region

  • We only provide an overview of methods here. If you want to really know the details about those algorithms, I invite you to consult (Nocedal and Wright 2006).

The Line Search Strategy

  • An algorithm from the line search class chooses a direction pk‚àà‚Ñùnp_k \in \mathbb{R}^n and searches along that direction starting from the current iterate xk‚àà‚Ñùnx_k \in \mathbb{R}^n for a new iterate xk+1‚àà‚Ñùnx_{k+1} \in \mathbb{R}^n with a lower function value.
  • After deciding on a direction pkp_k, one needs to decide the step length Œ±\alpha to travel by solving minŒ±>0f(xk+Œ±pk) \min_{\alpha>0} f(x_k + \alpha p_k)
  • In practice, solving this exactly is too costly, so algos usually generate a sequence of trial values Œ±\alpha and pick the one with the lowest ff.

The Trust Region Strategy

  • Here we construct a model function mkm_k that is similar to ff around xkx_k.
  • We acknowledge that mkm_k is decent approximation of ff only in some region.
  • The problem is then to find a candidate step length pp by solving minpmk(xk+p) \min_p m_k (x_k + p) where xk+px_k + p lies inside the trust region.
  • If candidate pp does not produce a value lower than f(xk)f(x_k), we must have had a too large trust region, shrink it, and do it again.
  • Usually the trust region is a ball ‚Äñp‚Äñ2‚â§Œî\Vert p \Vert_2 \leq \Delta, where Œî\Delta is called the trust region radius, but elliptical and box regions are possible.
  • A common definition of the model function is a quadratic of the form mk(xk+p)=f(xk)+pT‚àáf(xk)+12pTB(xk)p m_k(x_k + p) = f(x_k) + p^T \nabla f(x_k) + \frac{1}{2} p^T B(x_k) p where gradient and matrix B(xk)B(x_k) are evaluated at the current iterate, so the model function is in agreement to first order with ff at the current guess.
  • The matrix B(xk)B(x_k) is either the Hessian HfH_f or some approximation to it.

Line Search Methods: Which Direction to go?

Steepest Descent

  • The direction ‚àí‚àáf(xk)-\nabla f(x_k) is an obvious choice: Among all possible directions, along this one ff decreases most rapidly.
  • This claim can be verified using Taylor's theorem.
  • The steepest descent method is a line search method that moves along pk=‚àí‚àáf(xk) p_k = -\nabla f(x_k)
  • The step length Œ±k\alpha_k can be chosen in many different ways.
  • There are many other descent directions, steepest descent is but one of them.

Line Search Methods: The Newton Direction

  • Probably the most important descent direction.
  • In vector notation, the 2nd order taylor series approximation to f(xk+p)f(x_k + p) is f(xk+p)‚âàf(xk)+pT‚àáf(xk)+12pT‚àá2f(xk)p‚â°mk(p) f(x_k + p) \approx f(x_k) + p^T \nabla f(x_k) + \frac{1}{2} p^T \nabla^2 f(x_k) p \equiv m_k(p)
  • the Newton direction is obtained by finding the vector pp that minimizes mk(p)m_k(p), i.e. by setting the derivative of mk(p)m_k(p) to zero.
  • We obtain pkN=‚àí(‚àá2f(xk))‚àí1‚àáf(xk) p_k^N = -(\nabla^2 f(x_k))^{-1} \nabla f(x_k)
  • The newton direction is reliable if the discrepancy between truth and model mm is not too large at xkx_k.
  • The biggest drawback is the need to compute the Hessian. This can be difficult analytically at times, and overly expensive numerically.
optimize(rosenbrock, rosenbrock_gradient!, rosenbrock_hessian!, [0.0, 0.0], method = :newton)

Quasi-Newton Methods

  • In response to the difficulties of getting the Hessian, quasi-newton methods propose to approximate B(xk)B(x_k) with something similar to the hessian.
  • Taylors Theorem implies that ‚àá2f(xk+1‚àíxk)‚âà‚àáf(xk+1)‚àí‚àáf(xk) \nabla^2 f(x_{k+1} - x_{k}) \approx \nabla f(x_{k+1}) - \nabla f(x_k) and so we choose a BB matrix that mimics this property.
  • This leads to the secant condition Bk+1(xk+1‚àíxk)=f(xk+1)‚àí‚àáf(xk) B_{k+1} (x_{k+1} - x_{k}) = f(x_{k+1}) - \nabla f(x_k)
  • There are different ways to update the hessian in this way.
  • One of the best known is the BFGS method (after Broydon, Fletcher, Goldfarb and Shanno).
  • Those methods get the search direction by using BkB_k instead of the exact Hessian, i.e. pk=‚àíBk‚àí1‚àáf(xk) p_k = -B_k^{-1} \nabla f(x_k)
